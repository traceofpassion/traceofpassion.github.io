R로 블로그 만들기(1) - 깃(git)과 블로그다운(blogdown) 패키지

November 11, 2020 in 블로그, 시작하기

제가 처음 블로그를 만든 계기는 연세대학교의 이 모 교수님의 수업을 들으면서였습니다. 블로그를 만들어 게시물을 올리는 과제를 받았고, 수행하다보니 흥미가 생기고 이것저것 욕심이 생겨 예쁘게 꾸미고 기능을 추가하는 재미를 알았습니다. 그러나 마냥 행복하진 않았습니다. 아무래도 이 바닥이 다 그러하듯, 한국어 레퍼런스가 적어서 구글링을 해가며 찾는 과정이 지루할 때도 있었습니다. 블로그 메이킹 시리즈를 시작하는 이유는 혹시라도 누군가 저처럼 헤매는 이가 있다면 조금이라도 도움이 되고 싶은 마음에서입니다.

이 시리즈의 목표는 다음과 같습니다.

  1. R로 ‘그럴싸한’ 블로그를 만들어본다.
  2. 블로그에 있을 만한 기능을 추가해본다. (커스터마이징)

이 시리즈를 따라하고 싶은 사람은 다음과 같은 배경지식을 요구합니다. 만약 없다면 실습이 어려울 수 있습니다.

  1. R 기본기
  2. 마크다운과 R마크다운 문법
  3. 깃(git)의 개념

그럼 블로그 제작을 바로 시작합니다.

1단계. 깃(git) 초기 설정

1-1

  • 새로운 레포지토리를 생성합니다.

1-2

  • 레포지토리 이름은 원하는 대로
  • 블로그 작업 시 여러 개인정보가 코드에 포함되기 때문에 본인만 볼 수 있는 Private으로 설정합니다.

1-3

  • 깃 주소를 복사하여 추후 PC에 붙여넣기합니다.

1-4

저는 윈도우 운영체제를 사용하기 때문에 Git Bash 프로그램으로 커맨드 작업으로 깃 복사를 진행했습니다.

$ git clone "https://github.com/traceofpassion/TestBlog.git" # 1-3의 깃 주소
Cloning into 'TestBlog'...
warning: You appear to have cloned an empty repository. # 깃의 내용물이 없다는 경고. 무시합니다.

2단계. R Project 초기 설정

2-1

  • Rstudio를 실행한 후 새로운 프로젝트를 생성합니다.

2-2

  • 1-4에서 깃을 설치한 폴더를 경로로 설정하여 프로젝트를 생성합니다.

2-3

  • 해당 경로에 프로젝트 파일과 .gitignore 확장자 파일이 생성됩니다. 프로젝트 파일에서 블로그 작업을 하게 됩니다. 참고로, .gitignore 파일은 깃헙에 업로드할 때 불필요한 파일이 올라가지 않도록 설정하는 리스트입니다. 프로젝트 파일을 실행합니다.

2-4

콘솔 창에 아래의 코드를 차례대로 실행합니다.

install.packages("blogdown") # 블로그다운 패키지를 설치
blogdown::new_site(theme = "kakawait/hugo-tranquilpeak-theme") # 테마 설정
  • blogdown은 hugo라는 언어를 통해 R에서도 사이트 생성 작업을 할 수 있는 패키지입니다.
  • 다양한 테마를 고를 수 있는데, 그 중에서도 blogdown과 호환이 잘 되는 테마들이 있습니다. 이미지를 적재적소에 배치하여 블로그를 예쁘게 꾸밀 수 있는 kakawait의 tranquilpeak 테마를 선정했습니다.

2-5

  • 설치가 완료되면 우측 하단의 Viewer를 통해 생성된 블로그의 모습을 실시간으로 확인할 수 있습니다.
  • 홈 버튼을 클릭하면 로컬서버에서 블로그의 모습을 확인할 수 있습니다.

R로 블로그 만들기(2) - config.toml 설정하기

November 12, 2020 in 블로그, 시작하기

3단계. config.toml 설정하기

  • 아래 내용에 포함되지 않는 모든 코드는 별도의 수정을 하지 않았으니 default값을 유지합니다.

3-1

  • 프로젝트 파일 실행 후, 불러오기로 config.toml 파일을 실행합니다.
  • config은 configuration의 줄임말로, ‘구성’을 뜻합니다. 사이트의 여러 가지 구성 요소를 설정하는 파일입니다.

3-2

# I STRONGLY recommend you to use a CDN to speed up loading of pages.
# There is many free CDN like Cloudinary or you can also use indirectly
# by using services like Google Photos.
  • 가장 위에 있는 주석을 먼저 살펴봅니다. 이 테마의 저자는 CDN 방식을 사용해서 페이지를 업로드하기를 추천합니다. CDN은 Content Distribution Network의 줄임말로, 인터넷으로 여러 노드를 가진 네트워크에 데이터를 저장하여 제공하는 시스템입니다. 한 마디로, 구글 포토, icloud 등의 클라우드 방식으로 데이터를 제공하는 방식를 의미합니다. 저자는 속도 향상을 위해 CDN을 추천합니다. 저 또한 저자가 추천하는 Cloudinary를 통해 이미지 데이터를 제공합니다.

3-3

baseURL = "/"
title = "Test Blog"
# disqusShortname = ""
# googleAnalytics = ""
  • baseURL: 문서의 기본 url. 일단은 “/“로 설정.
  • title: 블로그의 제목. 필자는 “Test Blog"로 설정.
  • disqusShortname: 댓글 기능을 추가할 수 있는 disqus의 ID. 현재는 반영되지 않도록 주석 처리. 추후에 커스터마이징에서 다룰 예정.
  • googleAnalytics: 사이트의 트래픽을 측정할 수 있는 구글애널리틱스의 ID. 현재는 반영되지 않도록 주석 처리. 추후에 커스터마이징에서 다룰 예정.

3-4

[author]
  name = "나폴레옹"
  bio = "내 사전에는 불가능이란 없다."
  job = "황제"
  location = "프랑스"
  # Your Gravatar email. Overwrite `author.picture` everywhere in the blog
  # gravatarEmail = "thibaud.lepretre@gmail.com"
  # Your profile picture
  # Overwritten by your gravatar image if `author.gravatarEmail` is filled
  picture = "https://upload.wikimedia.org/wikipedia/commons/thumb/5/50/Jacques-Louis_David_-_The_Emperor_Napoleon_in_His_Study_at_the_Tuileries_-_Google_Art_Project.jpg/192px-Jacques-Louis_David_-_The_Emperor_Napoleon_in_His_Study_at_the_Tuileries_-_Google_Art_Project.jpg"
  • name: 블로그 저자의 이름
  • bio: 자신을 소개하는 문구
  • job: 직업
  • location: 사는 곳
  • gravatarEmail or picture: 프로필 사진을 설정하는 2가지 방법
    • gravatarEmail: gravatar(https://ko.gravatar.com/)에 회원가입 후 프로필 이미지를 업로드. 가입한 계정을 config.toml에 표기하면 자동 연동
    • picture: 본인이 설정한 이미지 업로드. 저는 CDN으로 구글에 이미지를 검색하여 그 주소를 연동했습니다.

3-5

[[menu.links]]
  weight = 1
  identifier = "github"
  name = "GitHub"
  pre = "<i class=\"sidebar-button-icon fa fa-lg fa-github\"></i>"
  url = "https://github.com/traceofpassion"
[[menu.links]]
  weight = 2
  identifier = "facebook"
  name = "Facebook"
  pre = "<i class=\"sidebar-button-icon fa fa-lg fa-facebook\"></i>"
  url = "https://www.facebook.com/hyeongsun.lee/"
  • github url: 자신의 깃헙 프로필 주소로 설정
  • linkedin url: 원래는 stackoverflow가 default값이나, 저는 잘 사용하지 않아 facebook으로 변경했습니다.
    • identifier와 name의 이름을 변경합니다.
    • Font Awesome Icons에 해당 사이트의 아이콘 이미지가 있는지 확인하고, 이름을 확인합니다. fa-(바꿀 아이콘의 이름)으로 변경합니다.
    • 해당 주소로 설정합니다.

3-6

# 블로그 소개글
  description = "나폴레옹의 테스트 블로그입니다."

# Copyright
[params.footer]
copyright = "<a href=\"https://github.com/traceofpassion\">나폴레옹</a>"
  • description: 블로그 소개글. 링크를 카카오톡에 공유할 때 블로그 이름 밑에 적히는 문구입니다.
  • copyright: 저작권 명시. 법적 효력이 있는지는 모르겠습니다.

3-7

blogdown::build_site()
  • 최종적으로 콘솔에 build_site() 함수를 통해 수정한 config.toml을 반영하여 사이트를 재구성합니다. build_site()는 게시글을 작성한 후에도 지속적으로 사용되니 기억해두면 좋습니다.

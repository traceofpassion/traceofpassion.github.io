R로 블로그 만들기(3) - Netlify로 사이트 배포하기

11월 12, 2020 in 블로그, 시작하기

4단계. Netlify로 사이트 배포하기

4-1.

$ cd TestBlog # 본인의 레포지토리 경로
$ git add .
$ git commit -m "first commit"
$ git push origin master
  • 지금까지의 내용을 커밋한 후, origin(깃헙)의 master 브랜치에 푸시합니다.

4-2.

  • Netlify는 정적 사이트의 호스팅과 백엔드를 제공하는 서비스입니다. 한 마디로, 제작한 블로그에 링크를 달아주는 역할을 하는 친구입니다.
  • GitHub 계정으로 Netlify의 회원가입을 마칩니다.

4-3.

  • 로그인을 마치고 나면 다음과 같은 화면이 나옵니다. 여기서 New site from Git을 클릭합니다.

  • Github을 클릭합니다.

  • Github 팝업창으로 이동하고, 다음과 같이 블로그 작업을 하는 레포지토리를 추가합니다.

  • 다시 Netlify 창으로 돌아와서 다음과 같이 해당 레포지토리를 설정합니다.

  • 브랜치는 master, command는 hugo, publish 디렉토리는 public으로 설정합니다.
  • master 브랜치의 public 폴더를 hugo로 빌드하여 배포한다는 의미입니다.

  • 사이트의 이름이 임의로 설정이 되었습니다. 사이트 이름을 변경하기 위해 Site settings를 클릭합니다.

  • Change site name을 클릭하여 사이트 이름을 변경합니다.

  • mytestblog로 사이트 이름을 변경했습니다. 사이트 이름은 고유한 값으로 mytestblog는 중복되어 이제 설정이 더 이상 불가능합니다.

4-4.

  • https://mytestblog.netlify.app 에 접속하여 블로그의 상태를 확인합니다.

4-5.

baseURL = "https://mytestblog.netlify.app/"

마지막으로, 이 주소를 config.toml의 baseURL에 반영하도록 합니다.

R로 블로그 만들기(5) - 게시글 작성(Posting)과 Cloudinary로 이미지 삽입하기

2020년 11월 15일 00:00 in 블로그, 커스터마이징

본격적으로 게시글을 작성(Posting)하는 방법과 Cloudinary를 활용하여 이미지를 링크 형태로 삽입하는 방법을 소개합니다.

7단계. 게시글 작성(Posting)

본격적으로 게시글을 작성하는 방법을 알아보겠습니다.

7-1.

  • 블로그 테마를 처음 설치했을 때 여러 기본 게시글이 있는 걸 확인할 수 있습니다. 그렇다면 게시글은 어떻게 작성할까요?
  • 경로를 확인합니다. 가장 상위 폴더로부터 /content/post 경로를 확인할 수 있습니다. 이 경로에 마크다운(.md) 확장자로 저장된 게시글이 위치합니다. 이 게시글은 테마를 제작한 kakawait가 남겨 놓은 좋은 샘플입니다. 이 샘플을 활용하여 원하는 형태로 게시글을 작성할 수 있습니다.

7-2.

샘플 중 하나인 Elements-showcase.md를 살펴보도록 하겠습니다.

---
categories:
- tranquilpeak
- features
date: "2015-05-28"
tags:
- html elements
- markdown
thumbnailImage: //d1u9biwaxjngwg.cloudfront.net/elements-showcase/vintage-140.jpg
thumbnailImagePosition: left
title: Elements showcase
---
  • 다음과 같은 YAML을 통해 여러 요소를 정의하고 있습니다. 간단히 설명을 하자면,
    • categories: 카테고리. 사이드 배너의 카테고리 버튼을 통해 관련 게시글을 쉽게 찾을 수 있습니다. 나열하는 순서대로 트리 구조가 적용됩니다(tranquilpeak 밑에 features가 위치함).
    • tags: 태그. 사이드 배너의 태그 버튼을 통해 관련 게시글을 쉽게 찾을 수 있습니다.
    • thumbnailImage: 썸네일. 홈 화면에서 게시글의 성격을 표현하는 이미지입니다. 저자는 CDN으로 웹의 이미지를 연결했습니다.
    • thumbnailImagePosition: 썸네일 이미지의 위치입니다.
    • title: 게시글의 제목입니다.
Check out how Tranquilpeak theme display well HTML elements (title, paragraph, blockquote, table and more..). It's simple and elegant.
< !--more-- >
  • 주목할 점은 YAML 아래의 < !--more-- >입니다.

  • < !--more-- > 위의 내용까지만 게시글의 부분을 홈 화면에서 제공하는 기능입니다. 이 기능은 RMarkdown(.Rmd) 파일에서는 적용이 되지 않아서 불편함이 있는데, 이를 개선하는 방법은 조금 복잡하여 추후에 따로 다루도록 하겠습니다.

7-3.

7-2 과정을 통해 샘플을 이해하는 과정을 살펴봤습니다. 샘플마다 YAML 구조가 조금씩 다르고 보여주고 싶은 기능이 여러 가지이므로, 필요할 때마다 하나씩 살펴가며 목적에 맞게 게시글을 작성할 수 있습니다. 이제 샘플을 블로그에서 제거합니다. 저는 기존 경로에서 가장 상위 폴더로부터 sample 폴더를 생성하고 여기로 샘플을 이동시켜 샘플을 보관하면서도 블로그에는 반영되지 않도록 조치하였습니다.

7-4.

이제 게시글을 생성하도록 하겠습니다.

blogdown::new_post(title = 'title1', ext = '.md')
  • blogdown::new_post()로 새로운 게시글을 생성합니다. Console에서 해당 코드를 실행하면 ./content/post/ 경로에 마크다운 파일이 생성되고 작업이 가능합니다.
  • 제목은 영문 작성을 추천합니다. 한글로 설정할 경우 정체 불명의 파일명이 만들어지기 때문입니다. 추후에 파일에서 YAML을 수정하면 한글 입력이 가능합니다.
  • RMarkdown 파일 생성을 원할 경우, 확장자를 .md 대신 .Rmd로 변경합니다.
---
title: 'md파일로 작성한 게시글'
author: ~
date: '2020-11-15'
categories:
- 블로그
- 마크다운
tags:
- 블로그
- 마크다운
keywords:
- 블로그 
thumbnailImage: //d1u9biwaxjngwg.cloudfront.net/elements-showcase/vintage-140.jpg
thumbnailImagePosition: left
metaAlignment: left
---
  • 7-2와 비슷하게 YAML 구조를 변경한 후 저장합니다.

7-5.

blogdown::build_site()

$ cd TestBlog # 본인의 레포지토리 경로
$ git add .
$ git commit -m "add md file"
$ git push origin master
  • 블로그를 빌드한 후, 지금까지의 내용을 커밋하여 origin(깃헙)의 master 브랜치에 푸시합니다.

8단계. Cloudinary로 이미지 삽입하기

이전 게시글에서 CDN을 언급한 적이 있습니다. 저자가 추천하는 방식은 Cloudinary였는데요, 제가 구글 포토도 사용해봤는데 Cloudinary가 사용하기가 편리했기 때문에 Cloudinary 사용법을 소개하고자 합니다. Cloudinary는 클라우드 베이스의 이미지와 비디오 서비스를 제공하는 회사입니다.

8-1.

사용법은 매우 단순합니다.

  1. Cloudinary 회원가입
  2. 이미지 업로드
  3. 해당 이미지의 링크 복사
  4. 필요한 위치에 복사하기

8-2.

썸네일 이미지를 Cloudinary 이미지로 설정하는 게시글을 작성해보겠습니다.

blogdown::new_post(title = 'clouidnary-example', ext = '.md')

title: 'Cloudinary 이미지 삽입하기'
author: ~
date: '2020-11-15'
categories:
- 블로그
tags:
- 블로그
keywords:
- 블로그 
thumbnailImage: https://res.cloudinary.com/traceofpassion/image/upload/v1605163814/post/blog_making/blog_thumbnail_ov1duz.jpg
thumbnailImagePosition: left
metaAlignment: left
  • 제 Cloudinary에 위치한 https://res.cloudinary.com/traceofpassion/image/upload/v1605163814/post/blog_making/blog_thumbnail_ov1duz.jpg를 삽입했습니다.

8-3.

blogdown::build_site()

$ cd TestBlog # 본인의 레포지토리 경로
$ git add .
$ git commit -m "add cloudinary example"
$ git push origin master
  • 블로그를 빌드한 후, 지금까지의 내용을 커밋하여 origin(깃헙)의 master 브랜치에 푸시합니다.

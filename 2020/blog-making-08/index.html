R로 블로그 만들기(8) - 부제목(Description) 사용하기

2020년 11월 19일 00:00 in 블로그, 커스터마이징

글의 제목 밑에 글의 내용을 간략하게 소개할 수 있는 부제목(Description)을 사용하는 방법을 소개합니다.

11단계. 부제목(Description) 사용하기

R로 블로기 만들기(5)에서 Markdown 파일에서 부제목을 어떻게 사용하는지 배웠습니다. 다만 RMarkdown에서 Markdown처럼 < !– more > 을 사용할 경우, 부제목 기능이 정상적으로 작동하지 않을 뿐더러, 본문에는 HUGOMORE42라는 텍스트가 삽입되어 이상한 결과를 낳습니다. 이러한 현상을 막기 위해 YAML 헤더에 Description 파라미터를 지정하고 이를 부제목으로 적용하는 방법을 소개합니다.

11-1.

먼저, 가장 상위 폴더로부터 ./themes/hugo-tranquilpeak-theme/layouts/_default/ 경로에 위치한 summary.html 파일을 메모장을 연결 프로그램으로 하여 실행합니다.

    {{ if .Params.Summary }}
      {{ .Params.Summary | markdownify }}
    {{ else }}
      {{ .Summary }}
  • Summary와 관련 있는 코드를 찾을 수 있습니다.
    {{ if .Description }}
      {{ .Description }}
    {{ else }}
      {{ .Summary | plainify | safeHTML }}
  • 코드를 위와 같이 Description과 관련 있는 코드로 변경합니다.

11-2.

Description이 정상적으로 작동하는 지 테스트해보겠습니다.

blogdown::new_post(title = 'desc-test', ext='.Rmd')
  • RMakrdown 형식으로 새로운 게시글을 생성합니다.
title: Description 테스트하기
author: ~
date: '2020-11-19'
slug: desc-test
categories: []
tags: []
keywords:
  - tech
description: Description 기능이 RMarkdown 파일에서도 정상적으로 작동하나요?
  • description 파라미터를 추가하여, 부제목을 직접 기입합니다.

11-3.

blogdown::build_site()

$ cd TestBlog # 본인의 레포지토리 경로
$ git add .
$ git commit -m "test descrption"
$ git push origin master
  • 블로그를 빌드한 후, 지금까지의 내용을 커밋하여 origin(깃헙)의 master 브랜치에 푸시합니다.

11-5.

Description의 정상 작동을 확인할 수 있습니다.
